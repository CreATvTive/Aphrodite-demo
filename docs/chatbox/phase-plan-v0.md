## A. 决策快照（最终值）

| 项 | 决定 |
|---|---|
| 运行形态 | 本地 Python 后端 + Web UI（localhost，前后端经 WebSocket 通信，消息格式 JSON） |
| 技术栈 | 后端 Python 3.10+；前端由执行器选型（建议 Vite + 任意轻框架），唯一硬要求：WebSocket + JSON 协议、能做逐字流出与多条消息切分 |

| 调用预算 | 测试期无上限；模型不可用时静默降级为纯 dynamics 运行（这本身是免费测试） |
| repo 策略 | 旧库 Aphrodite-demo 内新开 `app/chatbox/` 独立入口，检疫白名单制（见 B 节） |
| 场 v0 | 维度注册表制，出生 12 维（临时名见 C 节），软门控带宽 k=4，v0 竞争温度调平（门全开，行为等效普通 OU） |
| 动力学 | spring-damper + OU 噪声，1 tick/秒，软回归有界（禁硬 clamp）⏳ 参数接触时调 |
| 时间常数 | 快分量 ~10min，慢分量 ~3h ⏳ |
| 初始气质 | 手拍偏置：开放 -0.2、稳定 -0.1，其余 0 ⏳ |
| writer 格式 | B：自然语言一段（进日志）+ 尾附结构化增量（每维 -1~+1），单次每维幅度上限 0.3 |
| 感知信号 v0 | 消息间隔/沉默时长、时刻与昼夜、消息长度与节奏、会话开始结束事件、打字中状态（前端上报）。**架构为事件总线制**，后续加信号不改接口 |
| 感知入径 | 全部直接进 dynamics（手写映射表），不经 model call |
| 语言受体 v0 | 回复延迟分布（均值+方差）、长度倾向、切分倾向、标点语气松紧、逐字流出速率、表达压力（主动性）。prompt 侧受体走风格指令注入 + 禁元叙述合同测试 |
| 主动性 | 涌现制：表达压力累积量越阈触发（见 C.4），硬上限在场外：≤2 次/日、间隔 ≥6h、01:00–09:00 禁止（可配置） |
| 存储 | SQLite 单文件；每 60s 快照 + 事件即写；存 state 全池、门控状态、attractor、维度注册表、慢变量、对话、感知事件、writer 日志 |
| 慢变量 | v0 开启 baseline 漂移（周级时间常数）；维度注册表 v0 静态、池演化为 v0.x 首个重访点 |
| 备份 | 只防灾不回滚（B 方案）；慢变量漂移视为经历 |
| 反作弊 | 最小两习惯：验收测试**你本地亲手跑**；每次 review diff 必扫 `tests/` 目录改动。prompt 自省保留，输出仅作参考不作证据 |

## C. 核心机制规格（v0 冻结版）

### C.1 维度注册表（进合同）
- state 不是定长向量，是注册表：`{dim_id, 临时名, birth_time, strength, trigger_count, value, velocity}`
- 所有数值代码从注册表读维数，**任何地方不许硬编码维度数量**
- v0 注册表静态；池生长/萎缩是 v0.x 重访点

出生 12 维临时名（调试把手，不进任何 prompt 与语义层）：
能量、开放、稳定、朝向你、好奇、愉悦、紧张、疲惫、安全感、期待、沉郁、玩兴

### C.2 门控（v0 存根形态）
- 软竞争门（softmax 加温度），带宽 k=4
- v0 温度调平 = 门全开，行为等效无门控；温度作为配置项暴露，v0.x 拧
- **合同条款：受体只读门控后表达向量；dynamics 更新全池（门只遮蔽表达，不冻结演化）**

### C.3 writer 协议（进合同）
- writer 输出只移动 attractor（设定点），**永远不直接写 state**
- 单次每维移动幅度 ≤0.3（归一化空间），解析层负责截断
- 结构化增量解析失败时：丢弃增量、保留自然语言日志、attractor 不动（安全默认）

### C.4 表达压力（主动性的涌现实现）
- 场内新增一个累积量 `P_talk`：`dP/dt = g(朝向你, 期待, 沉默时长) − λ·P`（g 与 λ ⏳）
- `P_talk` 越阈 θ → 触发一次 writer 调用生成主动消息，`P_talk` 清零
- **硬上限层在此机制之外**：日次数、最小间隔、宵禁在出口处机械拦截，不参与动力学
- 定时器永久禁止

### C.5 prompt 侧受体与元叙述禁令（进合同）
- 受体值 → 一小段风格指令拼入 system prompt（只描述方式：句长、语气松紧，不许出现状态名与原因）
- 合同测试：回复文本不得出现对自身状态/指令的元叙述（"我现在心情…所以…"模式），用规则+抽样双检

## D. Phase 计划

每个 phase 的门 = 一个**你亲手运行并能看到的东西**。门不过，不开下一 phase。

### P1 骨架（无模型）
范围：维度注册表 + dynamics core（spring-damper+OU、软回归）+ 门控存根 + SQLite 持久化 + 最小 Web UI（只显示状态，无对话）+ 轨迹可视化页
任务卡：
1. 维度注册表与 dynamics tick 循环（验收：单元测试——10⁶ tick 无发散、无 NaN；任意维 attractor 阶跃后按时间常数收敛，误差带 ±20%）
2. 持久化与断线重启（验收：kill -9 后重启，state/注册表/轨迹完整，最大丢失 ≤60s）
3. 轨迹可视化（验收：浏览器实时看 12 维曲线与门控权重条）
**P1 门（你跑）**：开着可视化页放 1 小时——曲线在漂、有回归、不冻结、重启后接得上。

### P2 接模型
范围：provider 抽象层 + 对话调用（结构 A）+ writer 增量解析层 + 基础 prompt 侧受体（长度、语气）+ 对话 UI
任务卡：
4. provider 抽象 + 结构 A 调用（验收：配置切换 DeepSeek/Kimi 双跑通；输出两段解析成功率 ≥95%，失败走安全默认）
5. 解析层与 attractor 写入（验收：合同测试——增量越界被截断、非法格式不动 attractor、**无任何代码路径直写 state**）
6. 对话 UI + 风格注入 + 元叙述禁令测试（验收：50 条抽样回复零元叙述命中）
**P2 门（你跑）**：和她聊十轮，可视化页上看得到每轮对话把状态推动了，且她的话风随状态可感变化。

### P3 感知与受体全量
范围：感知事件总线（五信号）+ 手写映射表（信号→dynamics 输入项）+ 程序侧受体（延迟分布、切分、逐字流出）+ 扫维实验台
任务卡：
7. 感知总线与映射表（验收：注入合成事件序列，对应维度按映射响应，误差带内）
8. 程序侧受体（验收：给定受体向量，延迟/切分/流出速率的统计分布落在规格内；"犹豫=方差"可测）
9. 扫维实验台（验收：支持两种模式——钳制门强制上场扫单维、自由竞争扫联盟；一键生成盲测样本包）
**P3 门（你跑）**：盲测一次——固定同一句话，实验台生成 3 组不同状态下的回复样本，你能正确配对状态与样本组（配不对 = 该维受体失败，记录后继续）。另：对她沉默 2 小时，回来看轨迹里沉默留下的痕迹。

### P4 主动性与验收
范围：P_talk 累积量 + 硬上限拦截层 + 48h soak + 冻结/周期检测器 + 全合同测试收口
任务卡：
10. P_talk 与触发链路（验收：合成场轨迹下触发时机符合阈值定义；宵禁与频次上限在出口处不可绕过——**此上限代码位于受体与动力学模块外**）
11. soak 检测器（验收：方差塌缩与自相关峰值检测在注入的人工冻结/周期数据上召回 100%）
12. 48h soak 正式跑 + 全合同测试绿（你本地亲手跑）
**P4 门（你跑）**：48h 报告干净；期间她至少一次在合理时机主动开口且不烦人。**v0 完成。**
探测器（存在感判决）不在 v0 验收内——连续运行 ≥2 周后另行首验。

### v0.x 重访点登记（按序）
① 池演化（生长/萎缩，慢变量真身）② 竞争温度拧起（观察模式切换涌现）③ 时间常数与 OU 参数实验调优 ④ 重大感知事件升级 writer 调用 ⑤ 调用结构切 B ⑥ 初始气质重设

1. 本文档整体作为主计划投喂；每次只发一张任务卡，卡外改动一律拒收。
