## 执行摘要

Aphrodite 数学设计账本正确地识别了 14 个主要对象，并诚实地标注了它们的状态（已实现、仅影子、仅提案、负边界）。该账本本身是一个负责任的文档——它在大部分内容中恰当地陈述了其数学状态，无重大过度声明。

然而，上游语义解释路径（`input_interpreter.py` → 基于正则的观察器在 `store.py`）严重依赖硬编码的关键词列表和短语触发器。这是当前设计中最基本的数学风险：基于短语的解释向声称是定量的场流水线传递数据，但上游没有经过校准的语义模型。第一阶更新器（`updater.py`）数学上干净但脆弱——钳制隐藏了上游的不稳定性，累加求和放大了重复证据，带阈值在边界处产生不连续性。影子第二阶核心（`kernel.py`）结构良好，并经过良好测试，但目前仍未与运行时解耦，其物理隐喻存在被过度信任的风险。

系统最成熟的部分是模式层（`RelationalFieldState`、带有约束的确定性映射器）。最不成熟的是证据→信号→扰动链——它仍停留在 L1（启发式），但其输出被当作 L2（形式化工程规则）的输入。在将流水线推进到渲染器集成或优化之前，必须缩小这一鸿沟。

核心建议：（1）在进行进一步工程之前，记录上游解释的脆弱性；（2）为场更新器添加前钳制日志记录；（3）从 MotionCurve 中移除语义元数据泄漏；（4）在任何人尝试激活它之前，明确界定影子核心的退出标准。

### 成熟度等级定义

- **L0:** 隐喻/概念类比——无可实施的定量结构
- **L1:** 启发式规则或设计直觉——非正式但大致具有方向性
- **L2:** 具有有界实现的形式化工程规则——确定性、钳制、代码测试
- **L3:** 具有明确假设的可分析数学模型——可分析收敛性、稳定性、可辨识性
- **L4:** 具有明确目标、数据、验证和更新规则的可学习或可优化模型

### 3 个最有问题的对/簇

**簇 1：contamination_pressure ↔ contamination_resistance ↔ service_resistance**

单次污染检测同时触发三个轴上的累积、持久变化。由于衰减率不同（instant vs slow vs very_slow），每次污染事件后，抵抗随时间的变化形状是这些衰减曲线的卷积——无法仅通过查看当前值来单独识别。

**簇 2：structural_grip_pressure ↔ collaborator_layer_pressure**

两者具有相同的衰减曲线（fast, 0.45）和相似的上下文证据。如果没有明确的技术/项目上下文，这两个轴实际上无法单独识别。唯一的区分信号是 `technical_layer_needed`，它反相关这两个轴——如果技术层分类错误，系统会错误地减少握点压力并增加协作压力。

**簇 3：boundary_distance ↔ withdrawal_tendency**

它们是协同变化的——没有当前的扰动规则能将它们向不同方向移动。然而，它们的衰减率不同（boundary_distance=slow, withdrawal_tendency=medium），产生短暂的解耦：边界保持升高而退缩已经恢复——这可能只是建模伪影，也可能被误解为"冷漠但无退缩"。

## 第四部分：动态稳定性分析

### 4A：FieldStateUpdater

**公式：**

```
relaxed = current + decay_rate * (baseline - current)
next = clamp(relaxed + sum(delta), 0, 1)
```

#### 1. 有界钳制是否意味着稳定性？为什么/为什么不？

**不。** 有界钳制保证 BIBO（有界输入有界输出），而非李雅普诺夫意义上的稳定性。钳制对其范围内的值是一个无操作（线性）；它对接近边界的值是非线性的。在边界处，系统可以无限期地维持极限环：值被推至 1.0（钳制），衰减至 0.95（弛豫），下一轮被推回 1.0。输出保持在 [0,1] 内，但系统从不收敛至平衡。

零输入动力学在未钳制时是稳定的：只要 `decay_rate ∈ (0, 1]`，`current_{t+1} = (1 - decay_rate) * current_t + decay_rate * baseline` 保证收敛至基线。钳制不破坏稳定性，但掩盖了驱动信号是否在持续将系统推出自然平衡之外。

#### 2. 应记录哪些前钳制量以检测隐藏的不稳定性？

- `pre_clamp_value` = `relaxed + sum(delta)` — 每个轴、每个轮次
- `clamp_overflow` = `pre_clamp_value - next` — 钳制损失了多少
- `consecutive_saturation_count` — 连续饱和轮次计数
- `relaxation_direction` — 弛豫是否正在将状态拉向或拉离基线

#### 3. 重复同轴扰动下可能发生哪些故障模式？

- **钳制饱和/衰减循环（棘齿效应）：** 值被推至 1.0 → 钳制 → 衰减至 0.95 → 再次推至 1.0。系统在边界处振荡，但"看起来"稳定在 [0.95, 1.0] 的高值区间。
- **证据强度通胀：** 3 次 +0.10 = +0.30 加法，可能越过多个波段并触发硬约束。单轮净效应可能不合理地大。
- **衰减率错配：** very_slow 轴（如 presence_stability 为 0.05）累积所有历史；instant 轴（如 contamination_pressure 为 1.0）不累积。同一扰动在同一轮次对这两个轴的影响截然不同。

#### 4. 交替扰动下可能发生哪些故障模式？

- **符号追逐：** 一轮 +0.18，下一轮 -0.18，再下一轮 +0.18。系统在每个轮次大幅摆动，衰减阻尼交替信号。
- **波段的棘齿效应：** 如果高强度上升（+0.18）比低强度下降（-0.05）更频繁，系统会产生净漂移，即使"均值"扰动为零。
- **衰减引导：** 非常缓慢的衰减率会使先前的正扰动在衰减后仍处于高位，导致系统不对称地响应后续交替扰动。

#### 5. 扰动前弛豫还是扰动后弛豫有区别吗？二者有何不同？

**是的，有区别。** 当前顺序（弛豫后扰动）优先考虑扰动而非弛豫——扰动在衰减已将系统拉回基线后应用。

对比：
- **当前（弛豫后扰动）：** 从 0.80 开始，衰减率 0.30，基线 0.50 → 弛豫至 0.71 → 添加 +0.10 = 0.81
- **替代方案（扰动后弛豫）：** 从 0.80 开始 → 添加 +0.10 = 0.90 → 衰减率 0.30，基线 0.50 → 弛豫至 0.78

在 20 轮中，这一差异可累积至 ~0.50。

如果设计意图是"系统主要保持在基线附近，且扰动是暂时的"，则应优先采用先扰动后弛豫。当前顺序隐含地赋予了扰动比弛豫更强的权重。

#### 6. 同轴扰动应求和、饱和、平均还是竞争？

**建议：饱和。**

- 同号 → `max(|delta|)` — 不求和；取最强信号
- 异号 → `sum(delta)` — 净值计算，减后钳制
- 单轮上限为最大单次幅值（0.18）

理由：
- **求和** 会过度放大：三处提及边界压力各为 +0.10 = +0.30，可能越过三个完整波段
- **平均** 会稀释真正的累积效应：三次有效的 +0.10 提及应比一次 +0.10 更具影响力
- **竞争** 类似饱和，但未定义异号情况
- **饱和** 尊重"最强烈的信号取胜"原则，同时允许正负信号出价

#### 7. 在进一步信任此更新器之前，应添加哪些动态黄金案例？

- **DC-1：零输入回归基线。** 从 0.90 开始，零扰动。在 N 轮内验证衰减至 ≤ 基线 + ε。
- **DC-2：钳制饱和检测。** 持续高强度输入。验证 `consecutive_saturation_count` 递增，且前钳制值暴露漂移。
- **DC-3：交替符号对称性。** 交替 ±0.10 扰动。验证系统不以非对称方式漂移（除非衰减使然，且衰减应使然）。
- **DC-4：同轴求和上限。** 三个同号 +0.10 扰动。验证净效应 ≤ 0.18（饱和度上限），而非 0.30（求和）。
- **DC-5：带迟滞。** 从 0.19 → 0.21 → 0.19 振荡。验证带分类不会抖动（需迟滞守卫）。
- **DC-6：衰减率单调性。** 验证更快速的衰减率始终比更慢速的衰减率更快回归基线。

### 4B：影子 RelationalFieldDynamicsKernel

**公式：**

```
diag(M) * d²F/dt² + diag(C) * dF/dt + diag(K) * (F − B) = U(t)
```

#### 1. 相比一阶更新器，这在数学上引入了哪些行为？

- **惯性（速度记忆）：** 系统对速度具有"记忆"。即使力 U(t) 变为零，速度也会使系统继续运动。
- **超调：** 系统可以超过其平衡点。在欠阻尼情况下，值可以超过 1.0（超调上限将其钳制回）。
- **延迟响应：** 两个积分器（d²F/dt² → dF/dt → F）引入了相位滞后。系统对各点的响应并不立即成比例。
- **子步积分：** 通过自适应 dt 细分支持连续时间动力学。

#### 2. 场级惯性/速度带来哪些风险？

- **振荡：** 如果欠阻尼（阻尼比 ζ < 1），系统将围绕平衡点振荡。即使力变为零，振荡也会持续。
- **看不见的动量：** 内部门限 `F_tilde`（加速度积分器）是无界的。即使最终输出 F 被钳制，速度仍可能累积，导致突然的延迟超调。
- **多重平衡：** 如果 U(t) 持续非零，平衡偏移至 `F_eq = B + K⁻¹U`，而非 B。这意味着持续的输入流会创建一个远离基线的有效新平衡。
- **参数敏感性：** 二阶核心具有 53 个可调参数（M, C, K 各 10 个 + 10 个基线 + 3 个积分器参数），而一阶核心仅 23 个（10 个基线 + 10 个衰减率 + 3 个辅助参数）。调优空间大了一个数量级。

#### 3. 场级惯性可能与后续 MotionCurve 惯性或延迟产生哪些不良互动？

- **建设性延迟叠加：** 场核心延迟（相位滞后） + `initial_delay_sec` + 通道延迟因子。总延迟可能超过 3 秒，对响应式互动而言过慢。
- **双重积分器振荡：** 场核心振荡可能以不同频率与运动曲线缓动振荡产生跳动。拍频（频率差）可能在 0.5–2 Hz 范围内，肉眼可见。
- **抖动放大：** 场核心速度的小波动可能在运动曲线生成器中转化为逐帧幅度变化，尤其当位移缩放因子较大时。

#### 4. 在离开仅影子状态之前，必须保持哪些条件？

- **基线对齐：** B 必须从 `RelationalFieldState.baseline_numeric_value` 复制，而非独立指定。
- **无交叉轴耦合：** 10×10 隔离矩阵（全零非对角线元素）必须通过数值验证。
- **与一阶的等价性（退化参数选择下）：** 在 M→0、C=C_crit 的极限下，二阶核心应产生与一阶更新器数值上等同的输出。
- **无禁止导入：** 影子核心不得导入 behavior_affecting 模块。`behavior_affecting` 必须保持为 `False`。
- **退出标准需明确界定**，并且在满足之前需经人工审批。

#### 5. 哪些测试应证明直接交叉轴耦合不存在？

- **CA-1：对角线力隔离（10×10）。** 依次向每个轴施加力。验证只有目标轴发生变化；所有其他轴的 Δ < 1e-10。
- **CA-2：对角线加速度上限强制。** 向轴 0 施加大力，验证轴 1-9 的加速度保持为 0。
- **CA-3：掩码矩阵验证。** 验证 `self.config.mask_matrix` 在非对角线位置全为零，在对角线位置全为一。
- **CA-4：力交叉轴矩阵的数值验证。** 验证 `U_cross_axis = mask_matrix * U_raw` 等同于 `diag(U_raw)`（即无混合）。
- **CA-5：弹簧力隔离。** 向轴 0 施加偏移（F[0] ≠ B[0]），验证 K*(F-B) 力仅作用于轴 0，而非轴 1-9。
